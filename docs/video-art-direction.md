# 视频画风与创意导演规范

本项目不只生成讲稿，而是先生成“视频级设计系统”，再渲染成片。这个流程参考了 HyperFrames 的 agent-first 思路：先规划视频、设计镜头系统，再产出可验证的画面。

## 设计目标

- 亮色、活泼、轻松，有一点可爱感。
- 每个场景只讲一个观点。
- 每句关键讲解都要有对应的图形动作。
- 多用图、箭头、流程节点、检查卡、路线图，少用大段文字。
- 字幕保留，但字幕只辅助理解，不做主画面。

## 创意导演 Agent 输出

每个视频任务会经过 `BrightExplainerDirector`，它为每个分镜补充：

- `layout`：场景构图类型，例如聊天弹窗、词块传送带、路线图、流程管道。
- `visualPlan.metaphor`：这一幕的视觉比喻。
- `visualPlan.composition`：画面构图。
- `visualPlan.mainIllustration`：主图形。
- `visualPlan.motion`：动画动作。
- `visualPlan.flow`：流程箭头上的节点。
- `visualPlan.stickers`：贴纸式短提示。

## 当前样片镜头

| 场景 | 构图 | 目的 |
| --- | --- | --- |
| 开场 | 手机聊天 + 假论文卡 + 放大镜 | 3 秒内制造“AI 自信说错”的冲突 |
| 机制 | 词块传送带 | 解释“预测下一个词” |
| 误区 | 双路线地图 | 区分“像答案”和“有证据” |
| 高危 | 四向警示路牌 | 告诉观众哪些问题更容易翻车 |
| 流程 | 四步防幻觉管道 | 用箭头讲清楚降低幻觉的方法 |
| 工具 | 三问检查卡 | 给用户可立即使用的判断法 |
| 结尾 | 公式积木 | 总结 AI 生成 + 资料约束 + 人类把关 |

## 渲染规范

- 背景使用亮色渐变和柔和色块，不使用大面积暗色。
- 主元素使用粗描边、圆角、弹跳动画和轻微旋转。
- 箭头必须指向下一步，不能只做装饰。
- 字幕安全区固定在底部，不遮挡主图解。
- 每个场景至少包含一个主图形和一个流程/箭头/卡片结构。

## 参考来源

- HyperFrames：HTML-native、agent-friendly、deterministic video rendering。
- 短视频知识内容经验：静音观看友好、首屏冲突、低认知负担。
- 动画讲解视频原则：一个镜头只解释一个 takeaway。
