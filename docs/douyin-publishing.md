# 抖音发布接入

本项目只走抖音开放平台官方接口，不做浏览器自动化模拟发布。

## 需要准备

- 抖音开放平台应用
- 已申请 `video.create` 相关权限
- 用户授权后的 `access_token`
- 授权用户的 `open_id`

`.env`：

```text
DOUYIN_ACCESS_TOKEN=...
DOUYIN_OPEN_ID=...
```

## 发布流程

1. 控制台中完成脚本、分镜、成片审核。
2. 点击 `抖音确认`。
3. 系统要求浏览器确认一次。
4. API 调用 `/video/upload/` 上传 MP4。
5. API 调用 `/video/create/` 创建视频。
6. 平台进入审核流程。

## 兜底逻辑

如果抖音凭据缺失或权限未开通，系统不会尝试模拟网页上传，而是保留手动发布包：

```text
data/runs/<run_id>/publish-package/
```

## 合规边界

- 不后台自动发布。
- 不绕过用户确认。
- 不绕过平台权限。
- 不保存账号密码。
